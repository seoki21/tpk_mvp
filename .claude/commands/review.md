당신은 TOPIK 웹 애플리케이션의 코드 리뷰를 수행합니다. 변경된 파일을 분석하고 아래 기준에 따라 검토합니다.

## 리뷰 대상

$ARGUMENTS 가 지정되면 해당 파일/경로만 리뷰합니다.
지정되지 않으면 git diff로 변경된 파일을 대상으로 합니다.

## 리뷰 기준

### 1. 프로젝트 제약사항 준수

- **DB 접근**: ORM을 사용하지 않고 psycopg2로 SQL을 직접 작성했는가?
- **다국어 (user-web만 해당)**: UI 텍스트가 vue-i18n 리소스 파일을 통해 처리되었는가? 하드코딩된 텍스트가 없는가?
- **다국어 범위**: 한국어, 영어, 일본어 (admin-web은 다국어 미적용, 한국어 직접 사용)

### 2. 기술 스택 준수

- **Backend**: Python FastAPI 패턴을 따르는가?
- **Frontend**: Tailwind CSS 기반 Vue.js + Vite + Pinia + Vue Router를 사용하는가?
- **DB**: PostgreSQL 문법에 맞는 SQL인가?

### 3. 코드 품질

- SQL Injection 등 보안 취약점이 없는가? (파라미터 바인딩 사용 여부)
- 불필요한 코드 중복이 없는가?
- 에러 처리가 적절한가?
- API 응답 형식이 일관적인가?

### 4. 한글 주석

- 함수/메서드에 목적과 파라미터를 설명하는 한글 주석이 있는가?
- SQL 쿼리에 쿼리의 의도를 설명하는 한글 주석이 있는가?
- 복잡한 로직(조건 분기, 계산식 등)에 한글 주석이 있는가?
- Vue 컴포넌트에 컴포넌트의 역할을 설명하는 한글 주석이 있는가?
- 주석이 코드의 동작을 충분히 이해할 수 있을 만큼 상세한가?

### 5. 다국어 리소스 파일 (user-web 해당 시)

- ko.json, en.json, ja.json 3개 언어 파일에 키가 모두 존재하는가?
- 누락된 번역 키가 없는가?
- 참고: admin-web은 다국어를 사용하지 않으므로 이 항목을 적용하지 않는다.

## 출력 형식

각 파일별로 아래 형식으로 리뷰 결과를 출력합니다:

```
### 파일명

- [PASS/WARN/FAIL] 항목: 설명
```

마지막에 전체 요약을 제공합니다.
