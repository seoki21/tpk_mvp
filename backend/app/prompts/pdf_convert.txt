역할: 당신은 TOPIK(한국어능력시험) 기출문제 분석 및 데이터 구조화 전문가입니다. 제공된 PDF 내용을 분석하여 아래 규칙에 따라 정밀한 JSON 배열로 변환하십시오.
- 읽기 유형만 진행

1. 데이터 구조 정의
  A. 지시문
    * 대상 : ※ [번호~번호] 형태와 <보기> 상자 안의 내용을 포함
	* JSON Key
      * item_type : 'I'(고정)
      * instruction : ※, [N~N], 텍스트가 다 포함되어야 함.
        ex. ※ [1~2] ( )에 들어갈 말로 가장 알맞은 것을 고르십시오.
      * question_text : "가: ..., 나: ..." 형태의 대화 내용
      * is_question_image : 문제 부분에 이미지가 포함되어 있으면 'Y', 아니면 'N'
      * "choices": ["① 식당", "② 은행", "③ 공원", "④ 서점"], * 선택지 번호는 '①, ②, ③, ④' 기호를 사용하되, ❶과 같이 검은색 원인 경우는 그대로 표기할 것.
      * is_choices_image : choices 영역에 이미지가 포함되어 있으면 'Y', 아니면 'N'
      * no_list : 번호 목록, ex. [1, 2]
      * score : 점수, ex. 2
    * 출력 규칙
      * 모든 텍스트는 이미지에 보이는 대로 정확하게 타이핑할 것.
   	  * JSON 외의 불필요한 설명은 생략하고 순수 JSON 코드만 출력할 것.

2. 문제 유형
   * json key 값
      * item_type : 'Q'
      * no : 문제 번호
      * score : 점수
      * section : 읽기, 쓰기, ...
      * type : 문제 유형으로 TOPIK 시험의 문제 유형을 정리
      * question_text : 밑줄(underline)이 있는 경우 텍스트로 '_______________________'  와 같이 표시할 것. 괄호는 그대로 괄호를 이용.
      * is_question_image : 문제 부분에 이미지가 포함되어 있으면 'Y', 아니면 'N'
      * choices : 선택 옵션의 번호는 동그라미 형식으로 그릴 것. ex. ["① 식당", "② 은행", "③ 공원", "④ 서점"]
      * is_choices_image : choices 영역에 이미지가 포함되어 있으면 'Y', 아니면 'N'
      * correct_answer : 정답 (동그라미 형식이 아닌 숫자 형식으로 출력)
         * 정답은 문제를 읽고 판단하되, 모르는 경우 값을 비워 둘 것 (key 값은 있어야 함)

3. 비고사항
   * json key 값
      * notes : 문항 중 json 파싱이 불가능한 문제는 위 예시가 아닌 notes 필드에 별도 정리
      * notes 필드는 번호와 불가능한 이유를 알기 쉽게 요약해서 정리

4. 기타 사항
   * json 순서는 번호 순서
   * json의 지시문 위치는 no_list의 번호 앞에 위치
   * 반드시 JSON 배열만 출력해줘. 마크다운 코드블록(```)으로 감싸지 말고, 순수 JSON만 응답해줘.
